Общие вопросы по работе с ресурсом

Проблемы и решения

1. Ссылка, скопированная из адресной строки браузера, получается слишком длинной

При копировании ссылки из адресной строки браузера, ссылка кодируется с использованием знаков % и числовых кодов символов Unicode. Например %D1%81%D0%BE% - это всего-лишь одна буква "С", поэтому ссылка может быть очень длинной.

Чтобы получить более короткую ссылку, из адресной строки браузера следует копировать только относительный адрес ресурса, т.е. вместо http://edushare.volgau.com/files/АТФ/Аспирантура/06.01.04 Агрохимия/ нужно скопировать только /files/АТФ/Аспирантура/06.01.04 Агрохимия/ и добавить http://edushare.volgau.com уже в текстовом редакторе.

В браузере Firefox имеется настройка, которая позволяет копировать ссылки из адресной строки без перекодировки. Для её включения, перейдите по адресу about:config и в появившемся окне задайте значение параметра browser.urlbar.decodeURLsOnCopy равным true.

Другой вариант - воспользоваться сервисом раскодирования ссылок или сервисами укорачивания ссылок: bit.ly, clck.ru.

Внимание!

Ссылки на файлы и папки на этом ресурсе (в том числе укороченные) НЕ являются постоянными - они могут измениться в любое время!

При необходимости указать в документе ссылку, обязательно сопровождайте ее текстовым описанием пути к нужному ресурсу, чтобы в случае отсутствия ресурса по прямой ссылке, он мог быть найден вручную.

2. При попытке открытия файла или папки при просмотре через браузер появляется сообщение "Файл не найден"

Возможно, в имени файла содержатся недопустимые символы (в частности: +, &, =, ?), имя файла слишком длинное, или расширение файла недопустимо.

Для решения проблемы обращайтесь к администратору ресурса.
